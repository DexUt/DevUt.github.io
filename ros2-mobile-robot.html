ROS 2 Mobile Robot

I built a differential-drive mobile robot from the ground up in ROS 2, taking it from a CAD-accurate URDF to a physics-based Gazebo simulation with real-time TF frames, wheel odometry, and joystick/keyboard teleop. The stack includes a parameterized xacro description, a validated TF tree (map → odom → base_link → sensors), ros2_control with the diff_drive_controller, and RViz visualization (robot model, TF, laser scan, odometry). The result is a reproducible bring-up that launches the robot, spawns it into Gazebo, and drives it interactively while publishing ground-truth and fused odometry—ready for navigation and SLAM upgrades.

Key Details

  • Methods
    • Authored a modular URDF/xacro: base_link, left/right wheels, passive caster, set inertias/collisions from geometry; defined materials and sensor mounts.
    • Built a clean TF tree using robot_state_publisher and targeted static_transform_publisher frames (base_link → base_laser, base_link → imu_link), eliminating frame discontinuities and ensuring no TF loops.
    • Implemented ros2_control: joint_state_broadcaster + diff_drive_controller; tuned wheel radius, separation, velocity/accel limits, and PID gains to achieve straight-line and in-place rotation accuracy.
    • RViz2 bring-up: configured displays for RobotModel, TF.
    • Wrote launch files for one-command bring-up (description → controllers → Gazebo spawn → RViz).
    • Added teleop (keyboard/joystick) publishing geometry_msgs/Twist; recorded rosbag sessions and used rqt_graph/rqt_plot to validate topics and controller responses.
  • Tools
    • Software: ROS 2, xacro, tf2, ros2_control, gazebo_ros_pkgs, RViz2, teleop_twist_keyboard, colcon, rqt tools.
  • Outcome
    • Delivered a single-command bring-up that spawns the robot, starts controllers, and opens RViz—useful for demos and regression checks.
